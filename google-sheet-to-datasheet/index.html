用 AI 把 Google Sheet 規格表自動變成品牌 DataSheet PDF

從一個 Google Sheet 連結開始，讓 AI 讀取規格、下載產品圖片、建立模板，一路到產出 4 頁品牌 DataSheet PDF

2026-03-19 工具開發型
我用 AI 做了什麼 給 AI 一個 Google Sheet 規格表連結和 Google Drive 圖片資料夾連結，讓它自動建立 Camera DataSheet 生成系統並產出 PDF
沒有 AI 的話 需要手動在 InDesign/Illustrator 排版、逐一填入規格、調整圖片，估計 3-5 小時完成一份 DataSheet
最終成果 完整的 Camera DataSheet 生成系統（5 個新檔案）+ ECC100 4 頁品牌 PDF，整個過程約 10 分鐘
01

需求背景

我們已經有一套用 Python 自動生成 WiFi AP 和 Switch DataSheet 的系統，架構是 JSON 資料 → Pydantic 驗證 → Jinja2 HTML 模板 → Playwright PDF。現在公司推出了新的雲端攝影機產品線（ECC100），需要快速產出相同品牌風格的 DataSheet。

挑戰在於：Camera 的規格結構和 AP 完全不同（光學、影像、AI 分析 vs. 無線、天線、漫遊），需要全新的資料模型和模板，但規格資料散落在 Google Sheet 裡，產品圖片在 Google Drive 資料夾中。

!
既有系統架構：每個產品類別有各自的 models_*.py（Pydantic 模型）、templates/*_template.html（Jinja2 模板）、main_*.py（生成器），以及 data/*.json（規格資料）。

03

實作步驟

Step 1：讀取 Google Sheet 規格資料

Google Sheet 公開分享後，可以透過修改 URL 直接匯出 CSV，不需要任何 API 金鑰。關鍵是把 /edit 換成 /export?format=csv：

# 原始 URL
https://docs.google.com/spreadsheets/d/{SHEET_ID}/edit?usp=sharing

# CSV 匯出 URL
https://docs.google.com/spreadsheets/d/{SHEET_ID}/export?format=csv&gid=0

AI 會自動處理 Google 的 307 redirect，跟隨到實際的下載 URL 取得 CSV 資料。從 CSV 中，AI 擷取了 ECC100 的完整規格：

規格類別擷取到的資料
光學1/2.8" Sony STARVIS 5MP CMOS, Fixed 3.16mm, H:102° V:71.6° D:132°
影像5MP 2592x1944, 20FPS, H.265/H.264
AI 分析人員/車輛偵測、歷史搜尋、計數
音頻AAC, G.711, 內建麥克風
系統256GB, PoE 802.3af, 6.1W, IP67/IK10
Step 2：建立 Camera 資料模型

Camera 的規格結構和 AP 完全不同，所以需要全新的 Pydantic model。AI 參考既有的 models_engenius.py 架構，建立了 models_camera.py：

# models_camera.py 的核心結構
class OpticalSpec(BaseModel):     # 光學規格（感應器、鏡頭、視角、IR）
class ImagingSpec(BaseModel):     # 影像規格（解析度、幀率、壓縮）
class AISpec(BaseModel):          # AI 分析（物件偵測、臉辨、車牌辨）
class AudioSpec(BaseModel):       # 音頻（格式、麥克風、喇叭）
class HardwareSpec(BaseModel):    # 硬體（介面、電源、防護等級）
class StorageSpec(BaseModel):     # 儲存（容量、保留天數）
class ManagementSpec(BaseModel):  # 管理（瀏覽器、App、串流協定）
class CameraDataSheet(BaseModel): # 主模型，組合以上所有
Step 3：建立 HTML 模板與 CSS

Camera 模板保留了和 AP DataSheet 相同的品牌設計語言（藍色漸層頂列、EnGenius Cloud Logo、雙欄規格佈局），但內容區塊完全重新設計：

  • Page 1：Overview + 產品圖 + Features（和 AP 相同格式）
  • Page 2：光學/影像/AI/音頻/硬體/儲存規格（Camera 專屬）
  • Page 3：管理/環境/認證/尺寸/包裝內容
  • Page 4：Hardware Overview + 聯絡資訊 + 免責聲明
Step 4：從 Google Drive 下載產品圖片

產品圖片存在 Google Drive 的共用資料夾中。AI 使用 gdown 套件掃描整個資料夾結構，找到 ECC100 子資料夾，並下載所需圖片：

import gdown

# gdown 可以列出公開資料夾的所有檔案及其 File ID
folder_url = 'https://drive.google.com/drive/folders/{FOLDER_ID}'
files = gdown.download_folder(folder_url, skip_download=True)

# 取得 File ID 後，逐一下載需要的圖片
gdown.download(f'https://drive.google.com/uc?id={file_id}', output_path)

從 12 張 ECC100 圖片中，AI 挑選了兩張用於 DataSheet：

圖片用途選擇理由
ECC100_F-20_20240605.pngPage 1 主產品圖正面斜角照，展示完整外觀
ECC100_all_20240605_.pngPage 4 Hardware Overview6 角度全視圖，展示各面細節
Step 5：生成 PDF

執行 main_camera.py 完成 JSON → HTML → PDF 的完整流程：

python main_camera.py --json data/ECC100.json --save-html
04

踩坑紀錄

⚠ Google Sheet CSV Export 的 307 Redirect
!
症狀：直接 fetch CSV export URL 會得到 307 redirect 到 googleusercontent.com 的新 URL

解法：需要跟隨 redirect，對新 URL 再次發送請求。WebFetch 工具會自動偵測 redirect 並提示新 URL。

→
經驗法則：Google 服務的 export URL 幾乎都會 redirect，設計自動化流程時要預設處理 redirect 邏輯。
⚠ PDF 檔案被鎖定無法覆寫
!
症狀：重新生成 PDF 時出現 PermissionError: [Errno 13] Permission denied，因為前一版 PDF 正在 PDF 閱讀器中開啟。

解法：透過版本管理器讓系統自動遞增版本號（v1.1 → v1.2），生成不同檔名的 PDF 來避開檔案鎖定問題。

→
更好的做法：可在生成器中加入檔案鎖定偵測，自動加上後綴或提示使用者關閉檔案。
⚠ Google Sheet 資料不完整
!
症狀：Clean Data 第一個 tab 只有比較表等級的規格，缺少尺寸、重量、認證、包裝內容等 DataSheet 必要欄位。

解法：在 JSON 中將缺少的欄位填入 "—" 作為佔位符，讓 DataSheet 先能生成。待後續補充資料時，只需更新 JSON 檔案即可重新生成。

Takeaway

這個案例展示了什麼

用 AI 作為「系統擴展器」：當你已經有一套運作中的自動化系統，需要擴展到新的產品類別時，AI 可以理解既有架構、分析新需求的差異，然後一次性建立所有必要的新檔案。人只需要提供「資料在哪裡」和「要做什麼」兩個指令，AI 就能完成從資料擷取、模型設計、模板建立到最終產出的完整流程。

可移植性

這個方法適用於任何「既有模板系統 + 新產品類別」的擴展場景。前提是：

  • 已有一套可運作的模板生成系統作為參考架構
  • 新產品的規格資料可透過公開 URL 取得（Google Sheet、API、網頁等）
  • 產品圖片可透過公開 URL 下載

相同模式也能用於：產品型錄、比較表、報價單、Marketing 素材等需要從結構化資料生成品牌文件的場景。

如果你也要做，先問 AI 什麼
起手式 1 → 「我有一個 Google Sheet 規格表 [URL]，請讀取裡面的 [產品名] 資料，參考我現有的 [模板系統] 架構，幫我建立新的 [產品類別] DataSheet 生成系統」
起手式 2 → 「這個 Google Drive 資料夾 [URL] 裡有產品圖片，請找到 [型號] 的資料夾，下載適合做 DataSheet 的圖片」
