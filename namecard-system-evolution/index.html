01

需求背景

EnGenius Technologies 在全球有多個分公司（LA、APAC & India、EU、JP），每個分公司的名片格式略有不同 — 尤其 JP 有漢字名、東京本社地址邏輯。原本設計師用 Illustrator 逐張製作，效率低且容易出錯。

目標：從 Google Sheets 讀員工資料 → 自動套用 PDF 模板 → 輸出可印刷的名片 PDF，讓非技術人員也能直接在 Web 上操作。

03

Phase 1：PDF 生成 + 多地區 Tab UI

PDF 文字替換流程

用 PyMuPDF 的 redact → reinsert 模式：先用 add_redact_annot 標記要刪除的舊文字區域，apply_redactions 清除，再用 insert_text 寫入新文字。正面含 name / title / company / address / phone / email / website + QR Code，背面含公司名稱 + 網站。

字距追蹤（Letter Tracking）

PyMuPDF 不支援 letter-spacing，所以逐字元插入，每個字元的 x 座標累加 char_width + tracking：

TRACKING_PER_CHAR = {
    "name": 0.307,     # Medium 字體
    "title": 0.162,
    "company": 0.194,  # SemiBold 字體
    "address": 0.041,
}
多地區分頁 UI

前端用 Jinja2 的 namespace 技巧追蹤全域索引（跨 tab 的 checkbox 選取），每個 tab panel 獨立顯示該地區員工列表。

Phase 2：CJK 字體 + JP 背頁邏輯

逐字元切換字體解決日文亂碼

Manrope 字體缺少 CJK 字元（如「ー」U+30FC）。對每個字元檢查是否為 CJK，是的話切換到 NotoSansJP，tracking 歸零：

def has_cjk(ch):
    cp = ord(ch)
    return (0x3000 <= cp <= 0x9FFF or
            0xF900 <= cp <= 0xFAFF or
            0xFF00 <= cp <= 0xFFEF)

for ch in text:
    if has_cjk(ch):
        font, tracking = cjk_font, 0
    else:
        font, tracking = latin_font, field_tracking
JP 背頁東京本社地址邏輯

如果員工在東京本社（郵遞區號 108-0023），只顯示公司名 + 網站。其他辦公室則額外加印東京本社地址，網站 y 座標上移：

情境公司 y網站 y東京地址
一般（非 JP）89.88135.29無
JP 東京本社89.88135.29無
JP 其他辦公室89.88104.05135.29

Phase 3：CMYK 色差修復

⚠背面 redaction 填色與原始背景明顯色差
!
症狀：名片背面文字替換區域的藍色偏暗、偏綠，跟原始背景不一致

追蹤：用 get_pixmap 取樣 → redact 區域 #1b8fcf vs 原始 #0090d4。解析 content stream 發現模板用 0.95 0.26 0 0 k（CMYK），但 PyMuPDF 的 add_redact_annot(fill=...) 只支援 RGB。

根本原因：CMYK → RGB 轉換必然有色差，PyMuPDF 所有高階 API 都只支援 RGB。

解法：注入原生 CMYK 指令
# 1. Redact 時不填色
page.add_redact_annot(rect, fill=False)
page.apply_redactions(graphics=0)  # graphics=0 保留圖片

# 2. 注入 PDF 原生 CMYK 填色指令
c, m, y, k = 0.95, 0.26, 0, 0
pdf_y = page_height - pymupdf_y1  # PDF y 軸朝上
cmd = f"q {c} {m} {y} {k} k {x:.2f} {pdf_y:.2f} {w:.2f} {h:.2f} re f Q"
fitz.TOOLS._insert_contents(page, cmd.encode(), 1)
→
通用判斷法：content stream 裡 k = CMYK，rg = RGB。PyMuPDF 高階 API 一律用 RGB，要保持 CMYK 只能手動注入。
06

Phase 4：預覽 + RWD + 密碼登入

名片即時預覽

後端生成 PDF → page.get_pixmap(dpi=216) 轉 PNG → base64 → JSON 回傳前端。選 base64 而非暫存檔：不需清理、不需額外 route、不需考慮 Vercel /tmp 生命週期。

RWD 響應式表格
  • table-layout: auto — 瀏覽器依內容自動分配欄寬
  • 700px 以下隱藏「公司 / 地址 / 網站」，480px 以下再隱藏「電話」
  • 分公司 Tab 改為 pill 膠囊按鈕，選中整顆變藍
密碼保護

Flask session + Vercel 環境變數 APP_PASSWORD，空值時自動跳過（本機免密碼）：

APP_PASSWORD = os.environ.get("APP_PASSWORD", "")

def login_required(f):
    @wraps(f)
    def decorated(*args, **kwargs):
        if APP_PASSWORD and not session.get("authenticated"):
            return redirect(url_for("login"))
        return f(*args, **kwargs)
    return decorated
07

Phase 5：自動偵測 Google Sheets 分頁

原本分頁寫死在程式裡，新增地區就要改 code。解法：抓 Google Sheets /htmlview 頁面，用 regex 解析分頁資訊：

url = f"https://docs.google.com/spreadsheets/d/{SHEET_ID}/htmlview"
html = urllib.request.urlopen(url).read().decode()
matches = re.findall(r'name:\s*"([^"]+)".*?gid:\s*"(\d+)"', html)

for raw_name, gid in matches:
    label = raw_name.strip().rsplit("_", 1)[-1]  # "March. 2026_LA" → "LA"
→
風險控制：htmlview 不是官方 API，保留 _FALLBACK_TABS 備用設定。支援 _EXCLUDED_TABS（排除 HQ）和 _LABEL_ALIAS（APAC → APAC & India）。
!
命名規則：分頁標籤名用 任意文字_地區名稱 格式（如 March. 2026_TW），系統取底線後面的部分作為 Tab 名。
08

踩坑紀錄總覽

⚠apply_redactions 把 Logo 圖片也刪了

預設會刪除 redact 區域內的所有圖片。解法：永遠用 page.apply_redactions(graphics=0)

⚠PDF 座標系 y 軸方向與 PyMuPDF 相反

PyMuPDF y 朝下，PDF 原生 y 朝上。注入指令時：pdf_y = page_height - pymupdf_y1

⚠Icon 沒有隨長地址換行移動

地址兩行時 phone / email / website icon 需下移。用 get_drawings() 提取路徑 → 白色矩形蓋原位 → new_shape() 在新位置重繪 bezier 曲線。

⚠Vercel builds + functions 不能共存

vercel.json 裡只留 builds，不能同時有 functions。

Takeaway
