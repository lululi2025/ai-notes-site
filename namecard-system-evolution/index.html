01

需求背景

EnGenius Technologies 在全球有多個分公司（LA、APAC & India、EU、JP），每個分公司的名片格式略有不同（尤其 JP 有漢字名、東京本社地址邏輯）。原本設計師用 Illustrator 逐張製作，效率低且容易出錯。

目標：從 Google Sheets 讀員工資料 → 自動套用 PDF 模板 → 輸出可印刷的名片 PDF，讓非技術人員也能直接在 Web 上操作。

!
這個專案跨了 3 個 session 逐步進化，從最初的 PDF 生成腳本，到多地區 Tab UI，再到這次的預覽、RWD、自動偵測分頁、密碼保護。
02

技術選型

Python + Flask PyMuPDF (fitz) Google Sheets CSV Vercel Serverless qrcode + Pillow

03

功能演進紀錄

自動偵測 Google Sheets 分頁

原本分頁是寫死在程式裡的 SHEET_TABS 字典。使用者每新增一個地區分頁就要改程式碼。

解法：抓 Google Sheets 的 /htmlview 公開頁面，用 regex 解析 items.push({name: "March. 2026_LA", ... gid: "0"}) 自動取得所有分頁。

# 從 htmlview 解析分頁
url = f"https://docs.google.com/spreadsheets/d/{SHEET_ID}/htmlview"
html = urllib.request.urlopen(url).read().decode()
matches = re.findall(r'name:\s*"([^"]+)".*?gid:\s*"(\d+)"', html)
# → [("March. 2026_LA", "0"), ("March. 2026_APAC", "1291702676"), ...]

# 取底線後面的部分作為 Tab 名稱
for raw_name, gid in matches:
    label = raw_name.strip().rsplit("_", 1)[-1]  # "LA", "APAC", "JP"
→
風險控制：htmlview 的 HTML 結構不是官方 API，Google 隨時可能改。所以保留了 _FALLBACK_TABS 寫死備用設定，自動偵測失敗就 fallback。
名片即時預覽

使用者想在批次生成前先確認單張名片的效果。做法：後端用 PyMuPDF 生成 PDF → page.get_pixmap(dpi=216) 轉 PNG → base64 → JSON 回傳前端顯示。

選擇 base64 JSON 而非暫存檔 URL 的原因：不需要清理暫存檔、不需要額外 route、不需要考慮 Vercel /tmp 的生命週期。

RWD 響應式表格

原本表格是 table-layout: fixed + 固定 px 寬度，大螢幕看起來很窄、小螢幕溢出。改成：

  • table-layout: auto — 瀏覽器依內容自動分配欄寬
  • 移除 <th> 的固定 px 寬度
  • 700px 以下自動隱藏「公司 / 地址 / 網站」欄
  • 480px 以下再隱藏「電話」欄
密碼登入保護

用 Flask session + Vercel 環境變數 APP_PASSWORD 實作。不需要資料庫、不需要使用者管理。

# webapp.py
APP_PASSWORD = os.environ.get("APP_PASSWORD", "")

def login_required(f):
    @wraps(f)
    def decorated(*args, **kwargs):
        if APP_PASSWORD and not session.get("authenticated"):
            return redirect(url_for("login"))
        return f(*args, **kwargs)
    return decorated
!
如果 APP_PASSWORD 沒設（空字串），login_required 會自動跳過 — 本機開發時不需要輸入密碼。
04

踩坑紀錄

⚠ CMYK vs RGB 色差：背面藍色區塊顏色不對
!
症狀：名片背面文字替換後，redaction 填色的藍色跟原始背景明顯不同（偏暗）

根本原因：模板 PDF 用 CMYK 色彩空間（0.95 0.26 0 0 k），但 PyMuPDF 的 add_redact_annot(fill=...) 只接受 RGB。CMYK 轉 RGB 有色差。

# 解法：redaction 不填色 + 手動注入 CMYK 指令
page.add_redact_annot(rect, fill=False)
page.apply_redactions(graphics=0)

# 注入原生 CMYK 填色
c, m, y, k = 0.95, 0.26, 0, 0
cmd = f"q {c} {m} {y} {k} k {x} {pdf_y} {w} {h} re f Q"
fitz.TOOLS._insert_contents(page, cmd.encode(), 1)  # overlay=1
→
關鍵：PDF 原生座標系 y 軸朝上，PyMuPDF 朝下。轉換公式：pdf_y = page_height - pymupdf_y1
⚠ CJK 字體亂碼：日文字元用 Manrope 渲染
!
症狀：日文名片上「ー」（U+30FC 長音符號）等字元顯示為方框或亂碼

根本原因：Manrope 字體缺少 CJK 字元。解法是逐字元檢查，遇到 CJK 就切換到 NotoSansJP，同時 tracking 歸零（CJK 不需要字距調整）。

⚠ Redaction graphics=0 不能省略

page.apply_redactions() 預設會刪除 redact 區域內的圖片。名片背面整頁是藍色 + Logo，如果不加 graphics=0，Logo 圖片會被一起刪掉。

05

AI 協作分工

AI 負責
  • PDF 內部結構分析（content stream 解析、CMYK 操作符偵測）
  • PyMuPDF 底層 API 使用（_insert_contents、get_drawings、Shape API）
  • Google Sheets htmlview 逆向工程
  • Flask session + login_required 裝飾器
  • RWD CSS 媒體查詢 + 欄位隱藏策略
  • 前端 Tab UI pill 按鈕樣式重設計
  • Vercel 環境變數 + 部署指令
人負責
  • 提供名片模板 PDF + 設計規格
  • 截圖指出色差問題、icon 對齊問題
  • 決定功能優先順序（先預覽、再 RWD、再密碼）
  • Google Sheets 資料維護
  • Vercel Dashboard 設定環境變數
  • 最終品質確認（每次部署後驗收）
→
提問模式：症狀描述型 — 使用者提供截圖 + 描述症狀（「背景顏色不一樣」「icon 沒有對齊」），AI 從 PDF 底層結構找出根本原因。這種模式在 debug 視覺問題時特別有效。

Takeaway

這個案例展示了什麼

可移植性
直接複製 → 任何「從模板批次生成文件」的需求 — 證書、邀請函、報價單
可參考方法 → Google Sheets 作為非技術人員友好的資料來源 + 自動偵測分頁的 pattern
