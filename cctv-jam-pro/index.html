01

需求背景

目標是做一個讓人可以快速生成「看起來像真實監控畫面」的 AI 工具。使用情境是：輸入一個事件描述（例如「深夜便利商店搶劫」），工具自動生成符合 CCTV 視角、光線、位置的圖像。

這個工具的難點不是「生圖」本身，而是如何讓生成結果真的像 CCTV——有特定的攝影機角度、焦距、畫面比例，而不是普通的 AI 插圖。

→
關鍵洞察：CCTV 畫面的真實感來自鏡頭參數的精確控制，而不是靠描述詞堆疊。需要把「場景語意」和「攝影機物理參數」分開處理。
02

03

三階段生成流程

整個生成過程分成三個明確的步驟，每一步都由獨立的 Gemini API 呼叫完成：

使用者輸入：「深夜便利商店搶劫」
│
├── Step 1：analyzeTheme()
│   ├── 模型：gemini-2.0-flash
│   ├── 輸出：SceneData（結構化 JSON）
│   └── → location, time, character, camera params...
│
├── Step 2：synthesizePrompt()
│   ├── 模型：gemini-2.0-flash
│   ├── 輸入：SceneData
│   └── → 完整英文圖像提示詞
│
└── Step 3：generateCCTVImage()
    ├── 模型：flash-image（1K）或 pro-image（2K/4K）
    └── → base64 圖片 → localStorage 儲存
!
為什麼不直接一步生圖？因為中間的 SceneData 結構讓使用者可以手動調整參數（例如換攝影機角度、改時間），再重新合成提示詞，不必每次重新分析場景。
04

關鍵實作細節

Gemini Structured Output — JSON Schema 強制輸出格式

場景分析的輸出用了 Gemini 的 responseSchema 功能，強制模型回傳符合格式的 JSON，不會出現多餘文字或格式錯誤：

const response = await ai.models.generateContent({
  model: "gemini-2.0-flash",
  contents: prompt,
  config: {
    responseMimeType: "application/json",
    responseSchema: SCENE_DATA_SCHEMA,  // 定義好所有欄位型別
  },
});
→
這個做法比「請 AI 輸出 JSON 然後 parse」可靠很多——不需要 try/catch JSON.parse 失敗的情況，模型會直接保證格式正確。
攝影機參數設計 — 讓畫面真的像 CCTV

SceneData 裡除了場景語意，還有四個攝影機物理參數：

參數選項作用
cameraPlacementCeiling / Wall / Corner / Pole mounted決定拍攝位置與高度感
cameraTilt15° / 30° / 45° / 60° downward決定俯角大小，影響地面vs人物比例
subjectOrientationFacing / Side 20° / Side 45° / Diagonal back人物面對鏡頭的角度
fieldOfViewNarrow / Medium / Wide FOV模擬不同焦距鏡頭
Negative Prompt — 避免 AI 畫出「不像監控」的元素
// 如果人物朝向鏡頭，加入「不要眼神接觸」的負面提示
let eyeContactNegative = "";
if (data.subjectOrientation.includes("Facing camera")) {
  eyeContactNegative = "direct eye contact, staring at camera,";
}

// 固定的負面提示：移除鏡頭物件、去掉 UI overlay、避免魚眼
const prompt = `...
  - Negative: ${eyeContactNegative} camera device, fisheye, ui, text.`;
API Key 雙來源 — 環境變數 + 使用者輸入

為了讓工具在 AI Studio 平台和獨立部署（Vercel）都能運作，API Key 有兩個來源：

const envApiKey = process.env.API_KEY || '';        // Vercel 環境變數
const activeApiKey = userApiKey.trim() || envApiKey; // 使用者輸入優先

另外也支援 AI Studio 官方的 window.aistudio.openSelectKey()，讓平台上的使用者可以連結有付費方案的官方 Key 來使用 2K/4K 生圖。

05

踩坑紀錄

⚠ Bug 1：部署到 Vercel 後 App 無法運作
!
症狀：本機 npm run dev 正常，推上 Vercel 後 API 呼叫失敗或頁面白屏。

根本原因：Vercel 部署需要在平台後台手動設定環境變數 GEMINI_API_KEY，本機的 .env.local 不會自動上傳（也不應該上傳，避免洩漏）。

→
修復方式：在 Vercel Dashboard → Settings → Environment Variables 新增 API_KEY，重新部署後生效。
⚠ Bug 2：生圖時出現 429 Quota Exceeded
!
症狀：got status: 429 — You exceeded your current quota

根本原因：免費的 Gemini API Key 有每分鐘 / 每日呼叫上限，圖像生成模型的消耗量遠高於文字模型。

→
處理方式：等待配額重置（通常 1 分鐘），或換用付費方案的 API Key。工具本身在 UI 層也加了提示，遇到 403/429 錯誤時直接顯示說明並引導使用者切換到 1K 免費方案。
⚠ Bug 3：2K/4K 生圖出現 PERMISSION_DENIED (403)
!
症狀：切換到 2K 或 4K 後，呼叫 gemini-3-pro-image-preview 返回 403。

根本原因：Pro 影像生成模型需要 Google Cloud 專案啟用計費功能，一般免費 API Key 沒有這個權限。

→
處理方式：UI 上加了防守邏輯，在 AI Studio 環境偵測到未綁定付費 Key 時，直接攔截請求並提示使用者「立即連結官方 Key」，或引導切換回 1K。
06

Takeaway

AI 負責
  • 設計 Gemini structured output schema
  • 寫 prompt engineering（場景分析 + 提示詞合成）
  • Debug Vercel 環境變數問題
  • 處理 API 錯誤的 UI 提示邏輯
  • 實作 localStorage 資料管理
人負責
  • 定義產品功能（三步驟流程的設計）
  • 決定攝影機參數的維度（哪些參數影響 CCTV 感）
  • 判斷生成結果是否符合預期
  • 決定部署目標（Vercel）
這個案例展示了什麼

「把一個複雜任務分解成多個 AI 呼叫」比「用一個超長 prompt 一次搞定」更可控。分成「語意分析 → 參數化 → 生圖」三步，讓每個環節都有機會被人工介入修改，最終輸出品質更穩定。

另外，Gemini Structured Output（JSON Schema）是一個很值得記住的工具——只要你能描述清楚輸出的資料結構，就能讓 AI 輸出可以直接被程式碼使用的 JSON，省去大量 parse 防錯邏輯。

可移植性
直接可用 → 任何需要「輸入自然語言 → 轉結構化參數 → 生圖」的工具，都可以套用這個三階段架構
特殊條件 → 2K/4K 生圖需要 Gemini Pro 付費方案；免費 Key 只能用 1K
Vercel 部署 → 記得在 Dashboard 手動設定環境變數，.env.local 不會自動上傳
