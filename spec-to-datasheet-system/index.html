用 AI 建立產品 DataSheet 自動生成系統

從零開始，用 AI 逐步建立一套支援 WiFi AP、Switch、Camera 三大產品線的品牌 DataSheet PDF 自動生成工具，涵蓋資料驗證、模板引擎、圖片管理、版本控制的完整系統。

2025-12 ~ 2026-03 工具開發型（多輪迭代）
我用 AI 做了什麼 用 AI 建立完整的 DataSheet 自動生成系統：Pydantic 資料模型、Jinja2 HTML 模板、Playwright PDF 渲染、Google Drive 圖片管理、自動版本控制，支援 3 大產品線 8 款產品
沒有 AI 的話 每份 DataSheet 需要設計師用 InDesign 手動排版 3-5 小時，每次規格變更都要重新排版。8 款產品 x 多次改版 = 數百小時人工
最終成果 完整 Python 系統（20+ 檔案），已產出 17 份品牌 PDF。新增一款產品只需準備 JSON + 圖片，一行指令即可生成
01

專案目標

公司有多條網通產品線（WiFi AP、Switch、Camera），每推出新產品或規格更新時，都需要產出符合品牌規範的 DataSheet PDF。傳統做法是設計師用 InDesign 手動排版，耗時且容易出錯。

核心需求：建立一套自動化系統，讓行銷人員只需要維護 JSON 規格檔和產品圖片，就能一鍵產出專業品質的 DataSheet PDF。

痛點是什麼？ ├── 每份 DataSheet 手動排版 3-5 小時 ├── 規格更新要重新開 InDesign 改版 ├── 不同產品線格式不一致 └── 設計師是瓶頸，行銷無法自行產出 理想狀態？ ├── JSON 填規格 → 一行指令 → PDF 產出 ├── 版本自動遞增，歷史完整追蹤 ├── 三大產品線統一品牌風格 └── 行銷人員可獨立操作
02

系統架構

整個系統遵循 JSON Data → Pydantic Validation → Jinja2 HTML → Playwright PDF 的管線設計，每個產品類別有獨立的資料模型和模板，但共用版本管理和圖片管理模組。

系統架構總覽 │ ├── 資料層 (data/) │ ├── ECW215.json, ECW220.json ... ← WiFi AP 規格 │ ├── ECS1008P.json ← Switch 規格 │ └── ECC100.json ← Camera 規格 │ ├── 模型層 (models_*.py) │ ├── models_engenius.py ← AP: 無線/天線/漫遊 │ ├── models_switch.py ← Switch: 埠/交換容量/L2/L3 │ └── models_camera.py ← Camera: 光學/影像/AI │ ├── 模板層 (templates/) │ ├── engenius_template.html + CSS │ ├── switch_template.html + CSS │ └── camera_template.html + CSS │ ├── 生成層 (main_*.py) │ ├── main_engenius.py ← AP PDF 生成器 │ ├── main_switch.py ← Switch PDF 生成器 │ └── main_camera.py ← Camera PDF 生成器 │ ├── 共用模組 │ ├── version_manager.py ← 自動版本遞增 │ ├── google_drive_manager.py ← Google Drive 圖片下載 │ └── generate_datasheet.py ← 統一入口 CLI │ └── 輸出 (output/) └── DS_{型號}_v{版本}.pdf

04

三大產品線的差異

三大產品線雖然共用同一套系統架構，但各自有完全不同的規格結構和模板設計：

產品線已支援型號獨特規格區塊頁數
WiFi AP ECW115, ECW120, ECW215, ECW220, ECW230 無線頻段、天線場型圖、MIMO、漫遊 5-6 頁
Switch ECS1008P 埠規格、交換容量、L2/L3 功能、QoS 4-5 頁
Camera ECC100 光學/影像、AI 分析、音頻、防護等級 4 頁
WiFi AP 的 Pydantic 模型結構
class ProductDataSheet(BaseModel):
    # 基本資訊
    model_name: str           # "ECW220"
    product_line: str         # "Cloud Managed"
    overview: str             # 產品描述
    features: List[Feature]   # 最多 20 項

    # 技術規格
    standards: str            # "IEEE 802.11ax/ac/n/a/g/b"
    antenna: str              # "2x2:2 Internal"
    radio_chains: str         # "2x2:2"
    data_rate_2g: str         # "最高 573.5 Mbps"
    data_rate_5g: str         # "最高 1,201 Mbps"

    # 管理功能
    security: str             # "WPA3, WPA2, ..."
    roaming: str              # "802.11r/k/v"
    compliance: List[str]     # ["FCC", "CE", "IC"]
Camera 的 Pydantic 模型結構
class CameraDataSheet(BaseModel):
    optical: OpticalSpec      # 感應器、鏡頭、視角、IR
    imaging: ImagingSpec      # 解析度、幀率、壓縮格式
    ai: AISpec                # 人員/車輛偵測、臉辨、車牌辨
    audio: AudioSpec          # 編碼、麥克風、喇叭
    hardware: HardwareSpec    # 介面、PoE、防護等級
    storage: StorageSpec      # 容量、保留天數
    management: ManagementSpec # 串流、App、瀏覽器
05

核心功能拆解

版本管理系統

每次生成 PDF 時，系統會自動判斷是否需要遞增版本號：

  • 同一天重新生成 → 保持相同版本號（覆蓋）
  • 不同天重新生成 → 自動遞增 minor 版本（v1.2 → v1.3）
  • 版本歷史 → 完整記錄在 datasheet_versions.json，含日期和備註
{
  "ECW220": {
    "version": "1.9",
    "date": "02/25/2026",
    "history": [
      {"version": "1.2", "date": "12/01/2025", "note": "Initial version"},
      {"version": "1.8", "date": "02/18/2026", "note": "Updated specs"},
      {"version": "1.9", "date": "02/25/2026", "note": "Updated version"}
    ]
  }
}
Google Drive 圖片管理

產品圖片統一存放在 Google Drive 共用資料夾，系統支援多種 URL 格式的自動解析：

# 支援的 URL 格式
https://drive.google.com/file/d/{FILE_ID}/view
https://drive.google.com/open?id={FILE_ID}
{FILE_ID}  # 直接使用 File ID

# 自動下載流程
google_drive_manager.py:
  1. 解析 URL 取得 File ID
  2. 建立下載 URL
  3. 處理大檔案確認 Token
  4. 下載到 images/ 資料夾

圖片類型命名格式範例

硬體圖{Model}_hardware.pngECW220_hardware.png

# 2. 載入對應的 JSON 資料
# 3. 驗證資料完整性（Pydantic）
# 4. 套用對應的 HTML 模板
# 5. 渲染 PDF（Playwright）
# 6. 自動版本管理
# 7. 輸出 output/DS_ECW220_v1.9.pdf

08

Takeaway

這個案例展示了什麼

用 AI 不只是完成單次任務，而是建立一套可重複使用的系統。從第一份 AP DataSheet 開始，每次擴展新產品線時，AI 都能理解既有架構，快速複製模式並針對新需求做調整。最終形成的系統讓非技術人員也能自行維護和產出 DataSheet。

系統設計的關鍵決策
JSON over DB → 用 JSON 檔案而非資料庫，降低部署門檻，行銷人員可直接編輯
HTML over LaTeX → 用 HTML/CSS 做模板，任何人都能調整版面，不需學 LaTeX
模組化設計 → 每個產品線獨立模型/模板，新增產品線不影響現有系統
版本自動化 → 避免手動追蹤版本號，減少人為錯誤
如果你也要做，先問 AI 什麼
起手式 1 → 「我需要自動生成 [產品類型] 的 DataSheet PDF，規格資料在 [來源]，請建立完整的生成系統」
起手式 2 → 「我已經有 [既有產品] 的 DataSheet 系統，現在要擴展到 [新產品線]，請參考現有架構建立新的模型和模板」
起手式 3 → 「這個 Excel/Google Sheet [URL] 有產品規格，請讀取並轉換成我的 DataSheet 系統能用的 JSON 格式」
