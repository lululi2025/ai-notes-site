如何找回 Claude Code 的對話紀錄

Claude Code 把每次對話存在本機哪裡？怎麼跨專案搜尋、或找出特定專案的歷史 session？

2026-03-17 概念學習
我用 AI 做了什麼 請 Claude Code 說明自身的紀錄儲存位置，並讓它直接幫我翻出本專案的歷史 session 清單與內容摘要
沒有 AI 的話 需要自己找到 ~/.claude/ 資料夾、猜測資料夾命名規則、手動解析 JSONL 格式，整個過程容易卡關
最終成果 找到本機 15 個專案的紀錄資料夾，確認目前專案有 2 個 session，並取得每個 session 的對話摘要
01

情境背景

用 Claude Code 開發一段時間後，自然會好奇：之前那些討論、解法、決定，都存在哪裡？能不能像一般 chat app 一樣找回來？

這個問題有兩個層次：

  • 哪裡找：紀錄存在本機的哪個路徑？格式是什麼？
  • 怎麼搜：如何在多個專案之間用關鍵字搜尋，或縮小到某個專案查看？
!
Claude Code 不像 claude.ai 有雲端對話清單可以點選，所有紀錄都是本機 JSONL 檔案，需要用指令操作。
02

紀錄儲存架構

Claude Code 的對話資料集中存放在 ~/.claude/ 目錄下，Windows 的完整路徑是：

C:\Users\你的使用者名稱\.claude\
頂層結構
路徑用途
history.jsonl全域對話歷史，所有專案的完整紀錄都在這裡
projects/每個 git 專案各有一個子資料夾
memory/Claude 主動記下的跨 session 筆記（非完整對話）
plans/計畫文件
skills/自訂 skill 腳本
專案資料夾命名規則

每個專案資料夾以完整路徑轉換而來：把路徑分隔符號（/ 或 \）換成 -。例如：

C:\Lulu\Practice\[new]-cctv-gen-pro
→ 資料夾名：c--Lulu-Practice--new--cctv-gen-pro
專案內部結構
~/.claude/projects/c--Lulu-Practice--new--cctv-gen-pro/
├── 424e895c-f671-4031-b606-9388fd77716d.jsonl   ← session 1 對話紀錄
├── 540f6732-3c58-4642-8819-5e9722aaccb6.jsonl   ← session 2 對話紀錄
└── memory/                                        ← 此專案的 memory 筆記

每個 .jsonl 檔案對應一次開啟的 session，檔名是 UUID，修改時間即是最後互動時間。

03

搜尋方式整理

方法 A：在 Claude Code 介面直接 Resume

啟動 claude 後，可以在 session 選擇畫面恢復最近的對話。不需要下指令，是最快的方法——但只能找最近幾個 session，且限當前專案。

方法 B：用 grep 搜尋關鍵字

在全域 history.jsonl 裡搜尋文字：

# 搜尋全域對話記錄
grep "關鍵字" ~/.claude/history.jsonl

# 搜尋所有專案的 memory 筆記
grep -r "關鍵字" ~/.claude/projects/ --include="*.md"

# 搜尋所有 JSONL 對話檔案
grep -r "關鍵字" ~/.claude/ --include="*.jsonl"
→
JSONL 是純文字格式，每行是一個 JSON 物件，grep 可以直接搜尋。找到匹配行後，觀察前後的 "role" 欄位可以判斷是你說的還是 AI 說的。
方法 C：請 Claude Code 直接幫你查

這是本案例用的方式：直接告訴 Claude「幫我找這個專案的紀錄」，讓它執行 ls 和 Python 腳本解析 JSONL，回傳人類可讀的摘要。

→
對於不熟悉 JSON 格式的人，這是最省力的方式——你不需要自己寫解析腳本，AI 會幫你提取出有意義的對話片段。
方法適合情境限制
Resume 功能繼續剛做過的事只看得到最近幾個
grep 搜尋知道關鍵字，想跨專案找輸出是原始 JSON，不易閱讀
請 AI 幫查想知道某專案有哪些 session、內容是什麼需要 Claude Code 有 Bash 工具權限
04

實際操作示範

以本次對話的實際過程為例，操作流程如下：

Step 1：列出所有專案資料夾
ls ~/.claude/projects/

得到 15 個專案，確認哪個資料夾對應目前的 cctv-gen-pro 專案。

Step 2：列出該專案的 session 檔案
ls -la ~/.claude/projects/c--Lulu-Practice--new--cctv-gen-pro/*.jsonl

看到 2 個檔案，透過檔案大小（16KB vs 150KB）和修改時間初步判斷哪個更有內容。

Step 3：用 Python 解析並提取人類可讀的對話
cat [session.jsonl] | python3 -c "
import sys, json
for line in sys.stdin:
    obj = json.loads(line.strip())
    msg = obj.get('message', {})
    role = msg.get('role', '')
    content = msg.get('content', '')
    if role == 'user' and isinstance(content, list):
        for c in content:
            if c.get('type') == 'text':
                print(f'[USER] {c[\"text\"][:300]}')
    elif role == 'assistant' and isinstance(content, list):
        for c in content:
            if c.get('type') == 'text':
                print(f'[ASST] {c[\"text\"][:300]}')
"
!
JSONL 裡的 content 是陣列，每個元素有 type（text / tool_use / tool_result）。只取 type == "text" 就能過濾掉工具呼叫的雜訊，只看純對話。
結果摘要
Session大小內容
540f673216 KB只有一則：使用者輸入 /note，但當時沒有對話可整理
424e895c150 KB本次對話全紀錄（如何搜尋 Claude 紀錄 → 找出本專案 session）
AI 負責
  • 說明 ~/.claude/ 目錄結構與命名規則
  • 執行 ls 列出 15 個專案資料夾
  • 執行 Python 解析 JSONL，提取可讀對話
  • 判讀 session 內容並給出摘要
人負責
  • 提問：想找對話紀錄
  • 確認要找哪個專案
  • 判斷結果是否符合需求
05

Takeaway

這個案例展示了什麼

用「症狀描述型提問」——直接說「我想找紀錄」，而不是先去查文件——讓 AI 主導探索流程，最終用 3 個指令完成了手動可能需要 20 分鐘的翻找過程。

關鍵不在於你懂不懂 JSONL 格式，而在於把 AI 當成有 shell 工具的助理，讓它替你執行命令並翻譯輸出結果。

可移植性
直接可用 → 任何在本機跑 Claude Code 的人，路徑結構完全相同
特殊條件 → 需要允許 Claude Code 執行 Bash 指令（預設需要每次確認）
Windows 注意 → 路徑用 ~/.claude/（bash 語法），等同 C:\Users\你\\.claude\
如果你也要做，先問 AI 什麼

起手式 Prompt 1：找出所有專案

幫我列出 ~/.claude/projects/ 下的所有資料夾

起手式 Prompt 2：找特定專案的 session 摘要

幫我找這個專案（當前工作目錄）的 Claude Code 對話紀錄，列出有哪些 session、每個 session 大概在討論什麼
