靜態 HTML 雙語切換：
從 data-i18n 到 iframe 同步全記錄

一個展覽活動網站，兩個純 HTML 檔案，裡面全是 base64 圖片。
如何在不引入任何框架的情況下，加入中英文切換，並讓 iframe 裡的子頁面也跟著換語言？

2026-03-18 工具開發型 + 問題解決型
我用 AI 做了什麼 為 ISC West 2026 展覽活動網站（兩個純靜態 HTML 檔）加入中英文雙語切換功能，並解決 iframe 語言同步與 JavaScript 語法 bug
沒有 AI 的話 需要手動在 2–3MB 的 HTML 大檔中找到正確位置、逐一加入 data-i18n 屬性、撰寫 JS i18n 邏輯，以及自行設計 postMessage 跨 iframe 通訊機制，估計需要半天以上
最終成果 兩頁面皆支援點一下切換中英文；語言偏好存於 localStorage；iframe 子頁面透過 postMessage 三層機制自動同步；已推上 GitHub 並部署
01

需求背景

這是一個 ISC West 2026 展覽的工作人員行前說明網站，包含兩個純靜態 HTML 檔案：

  • index.html（主頁面）：行程表、素材清單、互動導覽入口
  • booth.html（攤位互動導覽）：嵌在 index.html 的 iframe 中，是一個點選式的攤位介紹

這兩個檔案各自嵌入了大量 base64 圖片，導致每個檔案都達到 2–3MB，無法用一般編輯器或 AI 的 Edit 工具直接修改（超過 token 上限）。所有大範圍文字替換都必須改用 Python 腳本執行。

!
限制條件：不能引入任何框架（Vue、React、i18next 等），只能使用原生 JavaScript。部署在 Cloudflare Pages（Direct Upload 模式），無伺服器端邏輯。
02

i18n 系統架構

採用「data 屬性 + JS 字典」的輕量 i18n 方案，不依賴任何外部函式庫。

HTML 端：標記要翻譯的元素

在需要翻譯的 HTML 元素上加入 data-i18n 屬性，值為翻譯 key：

<!-- 一般文字 -->
<h2 data-i18n="section.schedule">展覽行程</h2>

<!-- 包含 HTML 標籤的元素（例如 <br>）需加 data-i18n-html -->
<h1 data-i18n="hero.title" data-i18n-html="1">
  行前<br><span class="line2">懶人</span><br><span class="highlight">包</span>
</h1>
→
有無 data-i18n-html 屬性決定用 textContent 還是 innerHTML 設定翻譯值。前者更安全（防 XSS），後者允許 HTML 格式。
JS 端：翻譯字典與切換函式
// index.html 的翻譯字典（booth.html 中命名為 _b_i18n）
const _i18n = {
  zh: {
    'hero.title': '行前<br><span class="line2">懶人</span><br><span class="highlight">包</span>',
    'section.schedule': '展覽行程',
    'nav.booth': '互動導覽',
    // ... 約 57 個 key
  },
  en: {
    'hero.title': 'Pre-Event<br><span class="line2">Quick</span><br><span class="highlight">Guide</span>',
    'section.schedule': 'Schedule',
    'nav.booth': 'Booth Tour',
    // ...
  }
};

let _curLang = localStorage.getItem('lang') || 'zh';

function setLang(lang) {
  _curLang = lang;
  localStorage.setItem('lang', lang);

  // 遍歷所有標記元素，套用對應翻譯
  document.querySelectorAll('[data-i18n]').forEach(el => {
    const key = el.getAttribute('data-i18n');
    const val = _i18n[lang][key];
    if (val !== undefined) {
      if (el.getAttribute('data-i18n-html')) el.innerHTML = val;
      else el.textContent = val;
    }
  });

  // 更新切換按鈕文字
  const btn = document.getElementById('lang-toggle');
  if (btn) btn.textContent = lang === 'zh' ? 'EN' : '中文';
  document.documentElement.lang = lang === 'zh' ? 'zh-TW' : 'en';

  // 通知 iframe 子頁面
  const iframe = document.getElementById('booth-iframe');
  if (iframe && iframe.contentWindow) {
    try { iframe.contentWindow.postMessage({ lang: lang }, '*'); } catch(e) {}
  }
}

function toggleLang() { setLang(_curLang === 'zh' ? 'en' : 'zh'); }
document.addEventListener('DOMContentLoaded', () => setLang(_curLang));
切換按鈕樣式（踩過的坑）

按鈕初版 CSS 用了白色文字、白色邊框——因為 nav 背景是白色，所以完全隱形。最終改為藍色：

#lang-toggle {
  background: transparent;
  border: 1.5px solid #0070cc;   /* 不要用 rgba(255,255,255,...) */
  color: #0070cc;
  font-family: 'Space Mono', monospace;
  font-size: 11px;
  letter-spacing: 1.5px;
  padding: 5px 12px;
  border-radius: 4px;
  cursor: pointer;
  transition: all .2s;
}
#lang-toggle:hover { background: #0070cc; color: #fff; }
03

跨 iframe 語言同步：三層機制

booth.html 是以 iframe 的形式嵌入 index.html 的。語言切換時，需要通知 iframe 裡的 booth.html 同步切換。

單純在 DOMContentLoaded 時送 postMessage 並不可靠，因為 iframe 可能還沒載入完成。最終設計了三層機制，互相補位：

層次觸發時機作用
第一層：localStorage booth.html 自身載入時 直接從 localStorage 讀取語言，不依賴父頁面
第二層：boothReady 握手 booth.html DOMContentLoaded 後 向父頁面送出 {boothReady: true}；父頁面收到後再送語言
第三層：resizeBoothIframe iframe onload 事件 調整高度同時補送一次語言設定
booth.html 端：發送 boothReady
// booth.html 的完整 i18n 初始化
function _bSetLang(lang) {
  document.querySelectorAll('[data-i18n]').forEach(el => {
    const key = el.getAttribute('data-i18n');
    const val = _b_i18n[lang] && _b_i18n[lang][key];
    if (val !== undefined) {
      if (el.getAttribute('data-i18n-html')) el.innerHTML = val;
      else el.textContent = val;
    }
  });
}

// 接收父頁面的語言指令
window.addEventListener('message', function(e) {
  if (e.data && e.data.lang) _bSetLang(e.data.lang);
});

document.addEventListener('DOMContentLoaded', function() {
  // 第一層：讀 localStorage
  const lang = localStorage.getItem('lang') || 'zh';
  _bSetLang(lang);
  // 第二層：通知父頁面「我準備好了」
  if (window.parent !== window) {
    try { window.parent.postMessage({ boothReady: true }, '*'); } catch(e) {}
  }
});
index.html 端：接收 boothReady + resizeBoothIframe 補送
// 接收 boothReady，補送語言
window.addEventListener('message', function(e) {
  if (e.data && e.data.boothReady) {
    const iframe = document.getElementById('booth-iframe');
    if (iframe && iframe.contentWindow) {
      try { iframe.contentWindow.postMessage({ lang: _curLang }, '*'); } catch(ex) {}
    }
  }
});

// 第三層：resize 時補送
function resizeBoothIframe(iframe) {
  try {
    var h = iframe.contentWindow.document.body.scrollHeight;
    if (h && h > 400) iframe.style.height = h + 'px';
  } catch(e) {}
  try {
    var lang = typeof _curLang !== 'undefined' ? _curLang : (localStorage.getItem('lang') || 'zh');
    iframe.contentWindow.postMessage({ lang: lang }, '*');
  } catch(e) {}
}
04

踩坑紀錄

⚠ Bug 1：語言切換按鈕隱形
!
症狀：按鈕加上去了，但在頁面上完全看不見，點也沒反應。

根本原因：CSS 設定 color: rgba(255,255,255,0.85)、border: 1.5px solid rgba(255,255,255,0.5)，是白色在白色背景上。

/* ❌ 錯誤：白色文字＋白色邊框，在白色 nav 上消失 */
color: rgba(255,255,255,0.85);
border: 1.5px solid rgba(255,255,255,0.5);

/* ✅ 修正：改用明確的藍色 */
color: #0070cc;
border: 1.5px solid #0070cc;
→
預防：加按鈕時先確認父容器的背景色，避免使用半透明白色（常見於深色 nav 的設計，搬到淺色 nav 就悲劇）。
⚠ Bug 2：hero 大標題切英文沒反應
!
症狀：切換英文後，其他元素都換了，唯獨頁面最大的 h1 標題沒動。

根本原因：data-i18n="hero.title" 屬性已正確加在 h1 上，但 _i18n 字典的 zh 和 en 物件中都沒有 hero.title 這個 key。

// ❌ 字典缺少 hero.title key
const _i18n = {
  zh: { 'section.schedule': '展覽行程', /* ... */ },
  en: { 'section.schedule': 'Schedule', /* ... */ }
};

// ✅ 補上 hero.title（包含 HTML 結構，因為有 <br> 和 <span>）
const _i18n = {
  zh: {
    'hero.title': '行前<br><span class="line2">懶人</span><br><span class="highlight">包</span>',
  },
  en: {
    'hero.title': 'Pre-Event<br><span class="line2">Quick</span><br><span class="highlight">Guide</span>',
  }
};
→
預防：每加一個 data-i18n 屬性，立刻在字典兩個語言都加上對應 key，不要分兩步做。
⚠ Bug 3：booth.html 所有語言切換完全失效（根本原因）
!
症狀：index.html 切換英文後，嵌入的 booth.html 所有文字都不換，完全沒反應。即使直接開 booth.html，也無法切換。

根本原因：booth.html 的 <script> 區塊中，某個英文翻譯字串內有 ASCII 單引號（apostrophe，Unicode U+0027），但整個字串是用單引號包住的。這讓整個 script 標籤解析失敗，所有 JS 都不執行。

❌ 錯誤（apostrophe 截斷字串）
'desc.core1': 'Search what you're
looking for — with rules tailored
to your environment.'
// 在 you're 的 ' 處，JS 以為
// 字串結束了 → 語法錯誤
✅ 修正（改用雙引號或避開縮寫）
'desc.core1': 'Search what you are
looking for — with rules tailored
to your environment.'
// 最簡單：把縮寫展開
→
預防方法（三選一）：
  1. 英文翻譯統一用雙引號 "..." 包字串（不會與 HTML 屬性衝突）
  2. 在字串內的單引號前加反斜線：you\'re
  3. 用 Template Literal：`you're`
!
為什麼難排查：這種 JS 語法錯誤不會在 console 產生明確的 i18n 相關錯誤訊息，只會靜默失敗——所有函式都沒宣告成功，所有事件監聽器都沒掛上。表面看起來像「翻譯沒有觸發」，實際上是「整個 script 都沒跑」。
05

AI vs 人 分工回顧

AI 負責
  • 設計 data-i18n + JS 字典方案的整體架構
  • 撰寫 setLang / toggleLang / _bSetLang 函式
  • 設計三層 postMessage 同步機制
  • 用 Python 腳本對 2-3MB 大檔進行批次文字替換
  • 逐字分析 JS 語法錯誤根本原因（追查 Unicode apostrophe）
  • 撰寫所有 CSS 樣式（切換按鈕）
  • 執行 git commit / push 流程
人負責
  • 提供截圖說明「哪個地方沒有換成英文」
  • 決定翻譯措辭是否道地自然
  • 確認最終英文大標題為「Pre-Event Quick Guide」
  • 確認 Cloudflare 部署方式（手動上傳）
  • 提供贈品圖片路徑、素材圖片路徑
  • 確認內容正確性（工作內容描述、功能說明等）
關鍵轉折點

整個過程最關鍵的轉折是：直接用 Edit 工具改 2-3MB 的 HTML 檔案，超過 AI 的輸入限制。一開始試著用 Edit 工具逐行修改，失敗後改為「AI 寫 Python 腳本 → 執行腳本替換文字」的模式，這讓後續所有大量編輯都能順利進行。

第二個轉折是 booth.html 完全沒有語言切換反應——表面看起來像 postMessage 沒送到，但實際根因是整個 script 解析失敗。這是一個「靜默失敗」的 JS 語法 bug，透過逐步縮小範圍、最終檢查字串內容才找到。

06

Takeaway

這個案例展示的思路

「用 data 屬性標記 + JS 字典翻譯」是靜態 HTML 頁面最輕量的 i18n 方案，不需要框架，不需要 build step，只需要在 HTML 元素加一個屬性、在 JS 維護一個翻譯字典。

postMessage 跨 iframe 同步的關鍵是「不只送一次」——要預防 iframe 還沒準備好就收到訊息的競態條件（race condition），設計多個觸發點互相補位（localStorage + boothReady 握手 + onload 補送）。

靜態頁面需要 i18n → data-i18n + JS 字典，比 i18next 輕 100 倍
翻譯字串裡有縮寫 → 用雙引號包字串，或把縮寫展開
iframe 語言同步 → localStorage + boothReady 握手 + onload 三層補位
HTML 大檔（含 base64） → 改用 Python 腳本執行文字替換
可移植性

可以直接複製的情境：任何純靜態 HTML 頁面（單頁活動頁、說明文件、展示頁），不需要伺服器，不需要 npm install。

特殊條件：本案例因為有 iframe 子頁面需要同步，所以額外設計了 postMessage 機制。如果只有單一頁面，只需要 setLang 函式 + data-i18n 屬性即可。
